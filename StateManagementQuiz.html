State Management quiz 5/5/2019

  1. Can post requests be bookmarked? What are the problems?

    We cannot bookmark POST request. If POST is bookmarked that means that parameters exist only on the URL and all form parameter is lost at that time. And web server doesn’t know what to do with the request because form parameters was lost but this parameter was expected.

  2. What is the purpose of request dispatching?

    Purpose of request dispatching is to forward request to another servlet and ask it to handle problem itself. Or include the request to another servlet or JSP and then finish operation in the same servlet.

  3. What is the difference between redirect and request dispatch?

    If used redirect so web server sends back to client another WAP page and client use it to sent another request. If used request dispatch so the request handle in the server and then result send back to client.

  4. What is an attribute?

    It is a pair with name and key, where name is String and value is Object. Used in ServletContext, HttpServletRequest and HttpSession.

  5. What is the difference between attributes and parameters?

    Parameter only used for reading information, but attributes is can be used for reading and write information. Parameter has only String type, attributes is object type.

  6. What are dangers of using attributes?

    Is accessed in all servlets and requests in the application. And thread safe only in request attributes.

  7. What does it mean to say that http is stateless? Give an example of a stateful protocol.

    HTTP is stateless protocol it means that the every request doesn't keep any information about user or previous operations and states of this operations. But sometimes it needs to pass information of states between requests or user pages. For this case could be used cookies objects, that keeps in client side or sessions that used cookies to keep session ID. It possible to use hidden fields in HTTP forms.

  8. Give 5 different methods for maintaining state information (count each attribute scope as one method). Container managed state (3 scopes)

    1. Request scope is exist in servlet method that process this request and destroyed when servlet finished process this request.

    2. Session scope exist only while the browser is opened.

    3. When wab container closed the application scope is destroyed too.

    4. Cookies saved in client machine, deleted when browser closed.

    5. Fields on HTTP forms which hidden from user.

  9. How long does a session last?

    Is it possible to change the duration of session existence time.

  10. What is a cookie, and how long does a cookie last?

    Cookies are files in the client computers. Cookies use to keep useful information in files that used by server. Temporary cookies deleted when user browser was closed. But permanent cookies have different time of existence and depends on of parameter of this cookie. Cookies could be kept for 12 months or longer.

  11. What is the purpose of URL rewriting?

    URL rewriting using to make URL more readable and make the meaning of this URL string according the stricture and information of the pages.

  12. Why does the request attribute report ‘null’ for the maintaining state demo?

    If retuned result null so it is means that attribute by the given name is not exist. If is called method removeAttribute() for attribute it would have the same effect if is retuned null value of this attribute.
