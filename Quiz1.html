Servlet quis 5/3/2019

  1. What is the difference between a web server and a web container?

    Web server is a unit that manage web container and another components in this server, servlets, php.

  2. What is a servlet?

    Servlet is a component that processing requests from clients through web container.

  3. How do web servers and web containers interact with servlets?

    Web server get requests then use web container for finding and call servlets that processing the request.

  4. Who creates request objects?

    Client applications, http page for example.

  5. What are the states in the servlet lifecycle?

    Loading class, create instance, init, implementation of methods, destroy instance.

  6. Who calls init and when?

    Container call init after creation of certain servlet.

  7. Which of init, service, and doGet should you override?

    We should override doGet and doPost if we need to process request.

  8. In what sense are servlets multi-threaded?

    Web container create different threads from different application that send requests.

  9. What are the implications of this for servlet instance variables?

    Servlet not thread safe it means that different threads can change this variable or override different data in the same places of memory in the instance of servlet.
